# 分类系统重构计划

## 问题根源分析

经过深入分析代码，发现以下核心问题：

***

## 一、数据隔离机制现状

### 1.1 snippets 表的隔离机制

| 字段              | 本地片段             | 云端片段      |
| --------------- | ---------------- | --------- |
| `storage_scope` | `'local'` 或 NULL | `'cloud'` |
| `cloud_id`      | NULL             | 有值（云端ID）  |

**隔离逻辑** (`HomePage/index.tsx` 第 129-133 行):

```javascript
if (isLoggedIn) {
  result = result.filter(s => (s.storageScope ?? 'local') === 'cloud' || s.cloudId);
} else {
  result = result.filter(s => (s.storageScope ?? 'local') === 'local' && !s.cloudId);
}
```

**退出登录时**: `sync:clearCloudOnlySnippets` 删除所有 `cloud_id IS NOT NULL` 或 `storage_scope = 'cloud'` 的片段

### 1.2 categories 表的隔离机制

| 字段        | 说明                                       |
| --------- | ---------------------------------------- |
| `user_id` | `'local'` = 本地用户, `cloud_user.id` = 云端用户 |

**问题**: `CategoryManager.createCategory()` 的 `userId` 参数默认为 `'local'`！登录用户创建的分类实际上都变成了 `'local'` 用户的分类。

### 1.3 片段与分类的关联

* **关联字段**: `category_id` (存储分类 UUID)

* **冗余字段**: `category_name` (存储分类名称字符串) - 用于云端同步

***

## 二、核心问题清单

### 问题 1: 登录状态下创建片段，列表却没有

**根因**: `SnippetManager.createSnippet()` 创建片段时 **没有设置** **`storage_scope`**

* 片段 `storage_scope` 保持为 NULL

* 登录状态的过滤条件 `(s.storageScope ?? 'local') === 'cloud'` 无法匹配

* 导致新创建的片段被过滤掉

### 问题 2: 登录状态下，分类列表每个分类下永远是空

**根因 - 多重逻辑错误**:

1. **Sidebar 过滤逻辑错误** (`Sidebar/index.tsx` 第 105-114 行):

   ```javascript
   getSnippets = (catId, catName) => {
     return snippets.filter(s => {
       // ...登录状态过滤...
       return s.category === catName || s.category === catId;
       //                ^^^^^^^^ s.category 是 category_name（字符串）
       //                         ^^^^^^^^ catId 是 category UUID（永远不会相等）
     });
   }
   ```

2. **CategoryManager.getCategories() 的 count 统计问题**:

   * 它通过 `LEFT JOIN snippets ON c.id = s.category_id` 计数

   * 但云端片段的 `category_id` 可能为空（只有 `category_name`），导致不匹配

### 问题 3: 登录状态下，创建片段选定分类保存后不管用

**根因**: 云端同步下来的片段只有 `category_name` 没有本地 `category_id`

* `EditSnippetModal` 创建/更新片段时传递 `category.id`

* `dbSnippetToSnippet()` 返回的 `snippet.category` 从 `category_name` 读取

* 如果是云端片段，只有 `category_name` 有值，没有正确关联到本地 category\_id

### 问题 4: 分类的 user\_id 体系完全混乱

**根因**:

* `CategoryManager.createCategory()` 默认 `userId = 'local'`

* 登录用户创建的分类 `user_id` 错误地设置为 `'local'` 而不是 `cloud_user.id`

* 导致登录用户看不到自己的分类，本地用户的分类却显示给登录用户

***

## 三、重构方案

### 核心原则

1. **统一分类数据模型**: 一个用户一套分类，通过 `user_id` 严格隔离
2. **统一 category 引用**: 片段通过 `category_id` 关联分类，不再使用 `category_name`
3. **统一 storage\_scope**: 创建片段时根据登录状态正确设置 `storage_scope`
4. **修复过滤逻辑**: 分类过滤使用 `category_id` 而非 `category` 名称比较
5. **修复 user\_id 传递**: 创建分类时正确使用当前用户的 user\_id

***

### 步骤 4: 统一 category 引用 - 同步时正确映射 category\_id

**文件**: `frontend/src/main/services/SyncService.ts`

修改 `pullChanges()`，将云端分类名称映射到本地 category\_id:

```typescript
async pullChanges(): Promise<PullResult> {
  // ...
  for (const cloud of cloudSnippets) {
    // 查找本地对应的 category_id
    const category = db.prepare(
      'SELECT id FROM categories WHERE name = ? AND user_id = ?'
    ).get(cloud.category, cloudUserId);

    const categoryId = category?.id || null;

    db.prepare(`
      INSERT OR IGNORE INTO snippets
      (..., category_id, ...)
      VALUES (..., ?, ...)
    `).run(..., categoryId, ...);
  }
  // ...

```

### 步骤 5: 修复 Sidebar 过滤逻辑

**文件**: `frontend/src/renderer/components/Sidebar/index.tsx`

修改 `getSnippets()` 函数，使用 `category_id` 比较:

```typescript
const getSnippets = (catId: string, catName: string) => {
  return snippets.filter(s => {
    if (isLoggedIn) {
      if ((s as any).storageScope !== 'cloud' && !(s as any).cloudId) return false;
    } else {
      if ((s as any).storageScope === 'cloud' || (s as any).cloudId) return false;
    }
    // 使用 category_id 比较（优先），fallback 到 category 名称
    const snippetCategoryId = (s as any).categoryId;
    return snippetCategoryId === catId || s.category === catName;
  });
};
```

### 步骤 6: 修复 CategoryManager 的 count 统计

**文件**: `frontend/src/main/services/CategoryManager.ts`

修改 `getCategories()` 方法，count 查询需要正确处理云端片段:

```typescript
async getCategories(userId: string = 'local'): Promise<Category[]> {
  const categories = this.db.prepare(`
    SELECT c.*,
      (SELECT COUNT(*) FROM snippets s
       WHERE s.category_id = c.id
         AND ((s.storage_scope = 'cloud' AND ? != 'local') OR (s.storage_scope != 'cloud' AND ? = 'local'))) as snippet_count
    FROM categories c
    WHERE c.user_id = ?
    GROUP BY c.id
    ORDER BY c.created_at DESC
  `).all(userId, userId, userId);
  // ...
}
```
